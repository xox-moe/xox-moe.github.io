第四章 对象与类

类之间的关系

最常见的

  • 依赖 use-a
  • 聚合 has-a
  • 继承 is-a

还有 接口实现，关联和直接关联。 这个在UML中有讲 需要记忆这些关系分别对应的箭头的关系

更改器与访问器方法

mutator method and accessor method

大致意思就是 更改器方法会改变对象的状态 访问器方法不会改变对象的状态

例如

GreforianCalendar.add 方法是一个更改器方法 调用这个方法后 someDay的状态会改变

而新的java8中的LocalDate类设计的就不是这样了，他是 plusDay 会返回一个新的LocalDate对象 是指定天数后的

静态域和实例域

以前只是大概知道这两种的关系，这里使用书面的语言定义一遍

如果将域定义为static，每个类中就只有一个这样的域。而每个对象对于所有的实例域却都有一份自己的拷贝。

静态方法可以访问静态域 不能访问实例域

实例方法可以访问静态域和实例域

建议使用类名 而不是使用对象来调用静态方法。

方法参数

我个人觉得这块的知识比较重要 这决定了算法中的一些细节的写法，可以将更清晰的理解代码，从C++入门编程的我总是对Java的值传递方式有所疑惑。

程序设计语言中函数参数的调用方式大概有这几种

  • 值调用 call by value
  • 引用调用 call by reference
  • 名调用 call by name 这个出现在Algol这个语言中，现在已经很少见到了

在java中，参数有两种类型，

  • 基本数据类型（数字，布尔值）
  • 对象的引用

总的来说，java使用的是对象的引用的方式传递的参数，所以你可以改变引用的对象的值，而不能改变参数的引用的对象。

java程序设计语言对对象采用的不是引用调用，实际上， 对象的引用是按值传递的

下面总结一下java中方法参数的使用情况

  • 一个方法不能修改一个基本数据类型的参数
  • 一个方法可以改变一个对象的状态
  • 一个方法不能让对象的参数引用一个新的对象

初始化块

在一个类中，可以包含多个代码块，只要构造类的对象，这些块就会被执行。

初始化块分为静态初始化块和对象初始化块？？ 这块书中没有详细描述。

这种机制不是必须的，也不常见，通常会直接将代码放在构造器中。
